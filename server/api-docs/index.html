    Introduction

    Welcome to the PostIt API!

    This is a simple application that allows friends and colleagues create groups for notifications. This way one person can post notifications to everyone by sending a message once. The application allows people create accounts, create groups and add registered users to the groups, and then send messages out to these groups whenever they want.

    Authentication

    This authentication section is made up of the routes that registers a new user and login a user to their account.

    This endpoint creates a new user account on PostIt

    '/api/user/signup/'

    HTTP Request

    POST https://postit-webapp.herokuapp.com/api/user/signup

    Query Parameters

    Parameter Status Description
    username Required Username of the PostIt account creator.
    email Required Email of the PostIt account creator.
    password Required Password of the PostIt account creator.
    phone Required Phone number of the PostIt account creator.

    This endpoint allows an authenticated user create a new group

    '/api/create-group/'

    HTTP Request

    POST https://postit-webapp.herokuapp.com/api/create-group

    Query Parameters

    Parameter Status Description
    groupname Required groupname of the broadcast group.
    description Required description of the broadcast group.
    token Required authentication token of the group creator
    groupnames are all lower-case and no two groups can have the same name

    Errors

    This contains all the error response status code of the attempt to understand and satisfy the request. These codes are briefly explained after the Reason-Phrase. The Reason-Phrase is intended to give a short textual description of the Status-Code. The Status-Code is intended for use by automata and the Reason-Phrase is intended for the human user. The client is not required to examine or display the Reason- Phrase.

    The Kittn API uses the following error codes:

    Error Code Meaning
    200 OK - The request has succeeded
    201 Created - The request has been fulfilled and has resulted in one or more new resources being created.
    204 No Content - The server has successfully fulfilled the request and that there is no additional content to send in the response payload body.
    400 Bad Request -- The server cannot or will not process the request due to something that is perceived to be a client error (e.g., malformed request syntax, invalid request message framing, or deceptive request routing).
    401 Unauthorized -- The request has not been applied because it lacks valid authentication credentials for the target resource.
    403 Forbidden -- The server understood the request but refuses to authorize it.
    404 Not Found -- The origin server did not find a current representation for the target resource or is not willing to disclose that one exists.
    406 Not Acceptable -- You requested a format that isn't json.
    409 Conflict -- The request could not be completed due to a conflict with the current state of the target resource.
    413 Payload Too Large -- The server is refusing to process a request because the request payload is larger than the server is willing or able to process.
    500 Internal Server Error -- The server encountered an unexpected condition that prevented it from fulfilling the request.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
